# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch state.

## Reporting a Vulnerability

Please do not open public issues for security reports.

Report vulnerabilities by email to: `sebastiankurvers.apps@gmail.com`

Include:
- A clear description of the issue and impact.
- Reproduction steps or proof of concept.
- Affected commit/version and environment details.
- Any suggested mitigation if available.

## Response Targets

- Initial acknowledgement: within 3 business days.
- Triage decision (valid / needs more info / not a vulnerability): within 7 business days.
- Fix timeline: depends on severity and complexity; high-severity issues are prioritized.

## Disclosure

Please allow time for validation and patching before public disclosure.
Coordinated disclosure is preferred and appreciated.
